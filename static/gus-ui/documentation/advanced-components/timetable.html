Advanced Component - Time Table

Overview

Time Table is a useful components that let you display tasks or events among days and hours.

code

Attributes

Time Table has 3 attributes :
content (JSON Array) : correspond to an array of arrays which contains the tasks for each days, in normal order.
Each task is a JSON Object of type : {
  title: [string describing the task],
  startHour: [string correspond to the hour displayed where the task should start],
  startMinutes: [int from 1 to 59 that correspond to the number of minutes to add to the startHour],
  endHour: [string correspond to the hour displayed where the task should start],
  startMinutes: [int from 1 to 59 that correspond to the number of minutes to add to the endHour]
}

days (JSON Array) : array of string corresponding to the days at the top of the timetable

hours (JSON Array) : array of strings correspond to the hours displaying at the left of the timetable

In theory, the timetable lets you write anything you want in the hours and days attributes.
The tasks are displayed following the order of the JSON Array you're giving, which mean you just need to make sure that the days and tasks arrays are according to each other.
The startHour and endHour attributes of tasks can contain anything that is displaying in the hours.

CSS Parts

Time Table has 5 parts to customize :
timetable: correspond to the timetable itself
hours-holder: correspond to the column holding hours
days: correspond to the cells holding days
content-cell: correspond to the cells of the timetable grid
task: correspond to the tasks displaying on the timetable

Slots

Time Table doesn't have any slots.

Events

Time Table doesn't have any event to handle.
